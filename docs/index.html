get /api/v1/documents/{document_id}
Fetch a document (getDocumentById)
Returns information about the document specified by 'document_id'

Path parameters

documentId (required)
Path Parameter — Id of document to retrieve

Return type

Document

Produces

This API call produces the following media types according to the Accept request header; the media type will be conveyed by the Content-Type response header.
  • application/json; charset=utf-8

Responses

200

Document found

401

Unauthorized. Either the token provided is invalid or you do not have\naccess to the specific document

404

Document does not exists or you do not have access to it
Up 
post /api/v1/token
Fetch a new token via Oauth (createToken)
This will return a token that can be used for endpoints requiring authentication. The supported Oauth flows are 'client_credentials' and 'password'

Consumes

This API call consumes the following media types via the Content-Type request header:
  • application/x-www-form-urlencoded

Form parameters

grantType (required)
Form Parameter — The name of the Oauth grant used. Currently only 'client_credentials' and 'password' is supported
clientId (required)
Form Parameter — The Oauth client ID
clientSecret (required)
Form Parameter — The Oauth client secret
scope (optional)
Form Parameter — The scopes to associate with the token. The returned token will only be valid for endpoints that require the scopes specified in this parameter. Multiple scopes should be delimited by a space. default:
username (optional)
Form Parameter — The user's email address. Only used for 'password' grant
password (optional)
Form Parameter — User's password. Only used for 'password' grant

Return type

Token

Produces

This API call produces the following media types according to the Accept request header; the media type will be conveyed by the Content-Type response header.
  • application/json; charset=utf-8

Responses

200

Token successfully created

400

Request was invalid because of missing or invalid fields

401

Unauthorized. Usually because the client_id or client_secret is invalid
